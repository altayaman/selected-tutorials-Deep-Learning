                  Daniel LaCombe Mod David Cieslak 5 months ago

                  The way I got to the "8-bit binary vector" as I put it in the post, was to convert each string element first to Unicode (single integer value) and then to a binary string (of length 8). There are built-in functions for these operations in Python that you can hunt down.

                  I think those were the details you were interested in. Conceptually, it may be less powerful than other types of embeddings in that it may not explicitly capture semantic meanings, but one potential advantage is that it may be relatively insensitive to typos and things like that.

                  Unbeknown to me, character-embeddings have been around for a little while: https://papers.nips.cc/pape.... Not sure, however, how many researchers employ the embedding described above that Joey Prusa introduced me to.

                          Daniel LaCombe Mod Dan Ofer 5 months ago

                          In order to use the embedding layer in Keras, your text has to be tokenized -- which Keras also has functionality for. With most NLP problems, there is a vocabulary of words, but in my case, I was dealing with MPNs, each of which is unique. My logic was that this would explode the vocabulary which would be undesirable, and thus I just stuck with character-wise embedding. There will probably be functionality for character-wise within a couple weeks based on how quickly that library updates (if they deem it important enough).

                                  Daniel LaCombe Mod Dan Ofer 5 months ago

                                  I use the functional API almost exclusively now because it allows so much more flexibility. Have to be more careful with the naming conventions, as you may know, but much more natural IMO.

                                  To answer you question about the residual blocks, I used both residual blocks as well as skip connections (I think I'm using the terminology correctly). More specifically, if you look at the graph of the network, "activation_2" is summed as a residual with "activation_1" and passed on to "atrousconvolution1d_2". In addition, "activation_2" is used as a skip connection all the way to the last "merge" layer of the network.

                                  If by the first three layers you mean the four sets of convolutional filters I used to extract basic features, then in this case I only passed their residual to the next layer and did not include them in the skip connection to the last "merge" layer. I might have to go back and see if this improves performance.

                                  Check out the following link on more details about where I drew my inspiration: https://github.com/usernaam...

                                          Daniel LaCombe Mod Dan Ofer 5 months ago

                                          Thanks again for your questions, Dan. Each of these questions can be answered by looking at the network graph above. I only used two different modes for merging: sum and concatenation. Which one I used where can be inferred based on the output size. I'll still try and answer each in turn here though.

                                          > Did you try the residual connections on the initial "filter bank"? (i.e the first layer with multiple sized filters)?

                                          This I answered in the third block of my reply above. In the graph, "activation_1" represents the activation values of each of the initial filter banks concatenated together. As you can see in the graph, I did use this as a residual by summing it in a merge layer with "activation_2". However, I did not pass this as a skip connection all the way to the last merge layer ("merge_8") as I did with "activation_2".

                                          > Did you try attaching the input layer ("residual") together with each individually sized filter, or just their merge?

                                          If by input layer, you mean "input_1" in the graph above -- the actual data -- then no, I did not treat this as a residual. This, I think might constitute bad practice and would lead to overfitting of the data.

                                          Hope this helps.

                                            Daniel LaCombe Mod fooozhe 5 months ago

                                            The model just learns to output start and end indices, which are then used to extract the predicted MPN from the input string. Because it's a deep neural network, it's a bit of a black box. There are methods for peeking inside of it and getting a sense of what representations it's employing, but I didn't take the time for that here. So yeah, the end product is just a black-box model that has learned some good representations for indexing MPN codes. In other words, the regular expression is distributed across the network, but is learnt from the data rather than hard-coded.
